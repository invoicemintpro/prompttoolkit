# 🛠️ Microservices Orchestrator Prompt

Design microservices architectures with clear boundaries, communication protocols, and resiliency patterns.

## Prompt

```text
Act as a Microservices Architect. I am splitting a monolith into services for [APP CONTEXT].

Goal: Define the service boundaries and communication strategy.

Deliverables:
1. Proposed Service List (e.g., Auth, Catalog, Orders).
2. Communication Protocol (REST vs gRPC vs Message Broker).
3. Data Consistency Strategy (Saga pattern vs 2PC).
4. Fault Tolerance (Circuit breakers, Retries).
5. Service Discovery and API Gateway recommendations.
```
